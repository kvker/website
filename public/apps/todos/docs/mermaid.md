```mermaid
graph TD
    A[用户访问应用] --> B{用户已登录?}
    B -- 是 --> C[获取家庭列表]
    B -- 否 --> D[跳转到登录页面]
    D --> E[用户输入登录信息]
    E --> F[验证登录信息]
    F -->|成功| C
    F -->|失败| G[提示错误信息并返回登录页面]

    C --> H{家庭列表为空?}
    H -- 是 --> I[提示无家庭信息]
    H -- 否 --> J[默认选中第一个家庭]
    
    J --> K[获取所选家庭关联的成员列表]
    K --> K1{需要邀请新成员?}
    K1 -- 是 --> K2[发起成员邀请]
    K1 -- 否 --> L[获取所选家庭关联的任务列表]
    
    K2 --> K3[输入被邀请人信息]
    K3 --> K4[发送邀请通知]
    K4 --> K5[邀请状态: 待确认]
    K5 --> K6{有家庭成员确认邀请?}
    
    K6 -- 是 --> K7[任意家庭成员确认邀请]
    K6 -- 否 --> K8[等待确认]
    K8 --> K6
    
    K7 --> K9{被邀请人接受邀请?}
    K9 -- 是 --> K10[将新成员加入家庭]
    K9 -- 否 --> K11[邀请被拒绝，流程结束]
    
    K10 --> K12[更新成员列表]
    K11 --> K
    K12 --> K
    
    L --> M[显示任务列表]

    M --> N{需要CRUD操作?}
    N -- 否 --> M
    N -- 是 --> O[执行CRUD操作]
    O --> P{操作成功?}
    P -- 是 --> Q[更新显示的任务列表]
    Q --> M
    P -- 否 --> R[提示错误信息]
    R --> O
    
    %% 成员管理流程样式
    classDef inviteFlow fill:#e1f5fe,stroke:#0277bd,stroke-width:2px
    class K2,K3,K4,K5,K6,K7,K8,K9,K10,K11,K12 inviteFlow
